Notes: Testing and Debugging

Errors

  • There are generally 3 Types of errors when it comes to debugging: System, Programmer or User Error
  • There is little that we as programmers can do do fix system errors, so it is important to minimize the amount of errors that we the programmers make or the users make.
  • As we improve and do better as programmers it will lead to less errors both by us and the user. Good desgin leads to less errors overall.

Warnings

  • Warnings are good and bad.
  • Warnings are good because if the problem isn't large enough to crash the program and allows it to continue to run.
  • Warnings are bad beause they are allow the program to run with a defect in it.

Debugging and Testing

  • There are many tools to help us debug and to test our programs.
  • Some tools include linting tools, such as JS Lint
  • From my understanding JS Lint scans the code to see if it will agree with the rules of the code.
